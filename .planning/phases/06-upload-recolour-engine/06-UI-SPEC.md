**Element:** `.removal-progress` ‚Äî positioned absolute within `.car-ph`, replacing `.ph-upload`.

| `.rp-eyebrow` | JetBrains Mono, 10px, `letter-spacing: .22em`, `text-transform: uppercase`, `color: var(--accent)` | matches `.rk` in render-card |

### 3. Masked Car with Recolour

**Trigger:** `carUrl` is set (background-removed PNG dataURL).

**Behaviour:** The existing `fxLayers` mechanism in `stage.jsx` is unchanged. The only difference is that `carUrl` now points to a background-removed PNG rather than a raw photo. The `maskStyle` object (`WebkitMaskImage: url(${carUrl})`) naturally clips blend-mode layers to the silhouette.

**No new CSS classes required.** Existing `.car-base`, `.car-fx`, `.car-tint`, `.car-tone`, `.car-sheen` apply unchanged.

**The executor must NOT alter `fxFor()` return values for any finish.** The only addition is the metallic noise filter (see below).

### 4. Metallic Finish Noise Filter

**Requirement:** RCOL-06 ‚Äî metallic finish adds subtle grain noise to simulate flake depth.

**Implementation contract:** An SVG `feTurbulence` filter is defined once in the stage DOM and applied as a CSS `filter` property on the `.car-tint` layer only when `sw.finish === 'metallic'`.

**SVG filter definition** ‚Äî rendered once as a hidden `<svg>` inside `.stage`:

```html
<svg style="position:absolute;width:0;height:0" aria-hidden="true">
  <defs>
    <filter id="metallic-noise" x="0%" y="0%" width="100%" height="100%">
      <feTurbulence type="fractalNoise" baseFrequency="0.72" numOctaves="4" seed="2" result="noise"/>
      <feColorMatrix type="saturate" values="0" in="noise" result="grey"/>
      <feBlend in="SourceGraphic" in2="grey" mode="multiply" result="blended"/>
      <feComponentTransfer>
        <feFuncA type="linear" slope="1"/>
      </feComponentTransfer>
    </filter>
  </defs>
</svg>
```

**CSS contract:**

When `sw.finish === 'metallic'`, the tint layer receives:

```
filter: url(#metallic-noise)
opacity: 0.18   (added on top of the default metallic tint opacity from fxFor)
```

The noise filter renders only when `sw.finish === 'metallic'`. No other finish receives `filter: url(#metallic-noise)`.

**Visual threshold:** The effect must be subtle ‚Äî the grain should read as depth texture at normal viewing distance, not as visual noise. `baseFrequency: 0.72`, `numOctaves: 4` gives fine metallic flake texture. Do not increase `baseFrequency` above `0.80` or decrease below `0.60`.

**fxFor metallic case** ‚Äî add to `stage.jsx` switch statement (currently falls through to `default`):

```js
case 'metallic':
  return {
    tint: { background: c, mixBlendMode: 'color', opacity: .96 },
    sheen: { opacity: .22 },
    tone: { opacity: .18 },
    noise: true   // signals the renderer to apply the SVG noise filter
  };
```

### 5. Before/After Slider ‚Äî originalUrl Wiring

**Already built in prototype.** Phase 6 wires `originalUrl` into the "before" side.

**Contract:** In `stage.jsx`, when `baActive` is true and both `carUrl` and `originalUrl` are set:

- The "after" (wrapped) side: `carUrl` (background-removed cutout + fxLayers) ‚Äî unchanged
- The "before" (original) side: an additional `<img class="car-base car-base--original">` rendered at `z-index: 0`, using `originalUrl` as `src`, without any clip path (it sits behind the clipped wrapped layers)

**Visual contract:**

```
.car-base--original {
  position: absolute;
  inset: 0;
  width: 100%;
  height: 100%;
  object-fit: contain;
  z-index: 0;   /* behind .car-base (z-index:1) and fxLayers (z-index:2) */
}
```

The clip path (`inset(0 ${100-baPos}% 0 0)`) is applied only to `.car-base` (z-index 1) and `fxLayers` (z-index 2). The original image at z-index 0 is unclipped ‚Äî it shows on the left as the "before" side naturally as the clipped layers slide away.

**Before/after tags** (inherited from Phase 5): "Before" label on the left; "Wrapped" label on the right. No copy change.

**Activation:** The before/after toggle button (`.pill-btn` with `I.Compare` icon) is only enabled when `originalUrl` is set. When `originalUrl` is null, the button renders at `opacity: 0.4; cursor: default; pointer-events: none`.

**Accessibility:** The `.pill-btn` Compare button must carry `aria-label="Toggle before/after comparison"`.
