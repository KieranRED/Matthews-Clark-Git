    GET handler (hub challenge — webhook config step):
      - parse `new URL(request.url).searchParams`; read hub.mode, hub.verify_token, hub.challenge
      - if `mode === "subscribe" && token === process.env.WHATSAPP_VERIFY_TOKEN`: `return new Response(challenge, { status: 200 })`
      - else: `return Response.json({ error: "forbidden" }, { status: 403 })`

    POST handler — EXACT sequence (do not reorder):
      1. `const rawBody = await request.text();`  // MUST read raw body BEFORE any JSON.parse — Meta signs raw bytes (Pitfall 1)
      2. `const signature = request.headers.get("x-hub-signature-256") || "";`
      3. `if (!verifySignature(rawBody, signature)) return Response.json({ error: "invalid signature" }, { status: 403 });`
      4. schedule async work with after():
         ```js
         after(async () => {
           try {
             const payload = JSON.parse(rawBody);
             await handlePayload(payload);
           } catch (err) {
             console.error("[webhook][whatsapp][after-error]", err);
           }
         });
         ```
      5. `return Response.json({ ok: true });`  // 200 within 500ms regardless of downstream latency (FOUND-03)

    `verifySignature(rawBody, signature)` helper (FOUND-01):
      ```js
      const secret = process.env.WHATSAPP_APP_SECRET;
      if (!secret || !signature.startsWith("sha256=")) return false;
      const expected = "sha256=" + crypto.createHmac("sha256", secret).update(rawBody, "utf8").digest("hex");
      try {
        return crypto.timingSafeEqual(Buffer.from(expected, "utf8"), Buffer.from(signature, "utf8"));
      } catch {
        return false;   // length mismatch (Pitfall 6)
      }
      ```

    `handlePayload(payload)` helper (Pattern 4 — always loop):
      - `if (payload.object !== "whatsapp_business_account") return;`
      - for each entry in `payload.entry ?? []`, for each change in `entry.changes ?? []`:
        - `if (change.field !== "messages") continue;`
        - `const value = change.value;`
        - `const teamPhone = value?.metadata?.display_phone_number;`
        - `const contactName = value?.contacts?.[0]?.profile?.name ?? null;`
        - for each msg in `value?.messages ?? []`:
          - `await processInboundMessage({ wamid: msg.id, from: msg.from, to: teamPhone, type: msg.type, body: msg.type === "text" ? msg.text?.body : null, timestampMs: Number(msg.timestamp) * 1000, contactName });`
        - for each status in `value?.statuses ?? []`: `console.log("[webhook][status]", status.id, status.status);` (delivery receipts logged only in Phase 09)

    NOTE on FOUND-02: this phase only writes inbound; the store's `direction` column and message table already accommodate outbound (sent in Phase 11). No outbound send code here.
